## 1. 프로젝트 개요

- **이름**: 학식 말고 뭐 먹지? (campus-eats)
- **기술스택**: Next.js 16 + TypeScript + Tailwind + shadcn/ui, 카카오맵 API
- **디자인 원칙**: 정보 명확성 최우선, 브랜드 컬러 등 비주얼 완성도는 디자인 팀에게 맡김
  - 예: 내 위치 마커 vs 식당 마커는 색만 명확히 구분, 그 외 톤앤매너는 최소한만 적용

## 2. 디자인팀 시안과 다르게 간 부분 (+ 이유)

디자인팀 초기 와이어프레임은 **네이티브 앱 스타일**(하단 탭바, 홈/검색 화면 완전 분리, 구글 OAuth)이었으나, 아래와 같이 조정했습니다.

| 항목 | 디자인팀 시안 | 실제 구현 | 이유 |
|---|---|---|---|
| 인증 방식 | 구글 OAuth | 이메일/비밀번호 | PRD v2 기준 확인 결과, 디자인팀 시안이 PRD를 미반영한 것으로 확인됨. 이미 이메일/비번으로 구현 완료 |
| 화면 구조 | 하단 탭바 + 홈/검색 완전 분리 | 지도+목록 통합 화면 (네이버지도 스타일) | 웹 환경에서는 하단 탭바 방식이 자연스럽지 않다고 판단, 미채택. 다만 스크롤을 아래로 내리면 탭바가 나타나고 위로 올리면 사라지는 방식으로는 구현 가능하니, 필요하면 이 방향으로 다시 검토 가능 |

## 3. 디자인팀 시안에서 그대로 채택한 부분

- **요약카드 정보량**: 별점(리뷰수), 구역, 카테고리, 가격 — 영업시간과 사진은 추후 추가 가능하나 현재는 미구현
- **상세페이지 탭 구조**: 메뉴 / 리뷰 / 위치 3분할
- **위치 탭의 카카오맵 길찾기 버튼**

## 4. 실제 구현된 화면 구조

### 홈 + 식당 목록 (통합 화면)
- 지도 상단 고정 + 바텀시트 형태 목록 (네이버지도 스타일 참고)
- 마커 ↔ 목록 항목 연동 (하나 선택하면 다른 쪽도 하이라이트)
- 마커 클릭 시 요약 카드 노출
- 내 위치는 `navigator.geolocation`으로 표시 (마커 색으로 식당과 구분)
- **[신규 요청] 현재 내 위치로 돌아가기 버튼**: 지도를 스크롤/이동한 후 다시 내 위치로 돌아올 수 있는 버튼 필요 (보통 지도 우측 하단에 위치, 네이버지도·카카오맵의 GPS 버튼 참고). 아직 개발 반영 전이라 디자인·개발 양쪽에서 함께 챙겨야 할 항목입니다.

### 검색
- 대상: 식당명 + 메뉴명
- 트리거: 버튼 클릭 또는 Enter (실시간 자동완성 없음) — 검색창에 별도 자동완성 드롭다운 UI는 불필요
- 결과 없음 상태 문구: "검색 결과가 없어요. 새로운 식당을 제보해보세요!"

### 필터 시스템
- **구역**: 정문 / 상대 / 예대 / 후문 / 공대쪽문 (5개, 학교 특성에 맞춰 이름 기준으로 분류)
- **카테고리**: 한식 / 중식 / 일식 / 양식 / 분식 / 카페 (6개)
- **가격대**: ~5천 / ~1만 / ~2만 / 2만~ (4구간, 메뉴 최저가 기준)
- **제휴이벤트**: 체크박스 (진행 중인 이벤트가 있는 식당만 필터링)
- 위 4개 필터는 서로 독립적으로 AND 조합되며, "전체" 선택 시 해당 필터는 조건에서 제외되는 토글 로직

### 정렬
- 평점순(기본값, 평균 별점 내림차순) / 거리순(GPS 거부 시 학교 정문 고정 좌표로 대체 계산) / 가격순(최저가 오름차순)
- 정렬 옵션 UI(드롭다운 또는 탭 형태) 필요

### 식당 상세페이지
- 메뉴 / 리뷰 / 위치 3개 탭 분리
- 리뷰 탭: 별점(1~5), 한줄평 목록 + 본인 작성 리뷰는 수정/삭제 버튼 노출
- 즐겨찾기 토글 버튼 (목록/상세 공통)

### 마이페이지
- 내 즐겨찾기 목록 (식당명 클릭 시 상세 이동)
- 내 제보 현황 (상태: 대기 / 승인 / 반려)

### 헤더 (전 페이지 공통)
- 순서: 로고 - 닉네임 - 마이페이지 - 관리자 - 로그아웃 (로그아웃은 오른쪽 끝 단독 배치)

### 제휴이벤트 배지
- 진행중 / 만료 / 시작전 3가지 상태로 구분 표시 (별도 on/off 스위치 없이 시작일·종료일 기준 자동 판별)

## 5. 이번 MVP에서 제외한 기능 (디자인 작업 시 참고)

아래 기능은 시안에 있더라도 이번 범위에서는 구현하지 않았습니다.

- **영업시간 필드**: PRD 범위 밖으로 확인되어 제외 (팀과 추후 논의 예정)
- **알림 기능**: 실익이 낮다고 판단, 미구현 결정
- **이메일 인증 / 비밀번호 찾기(이메일 발송)**: PRD 범위 밖. 대신 회원가입 시 이메일 형식 검증을 최소한으로 강화(오타성 이메일 방지)하고, 비밀번호를 잊은 경우 로그인 페이지의 "비밀번호를 잊으셨나요? 문의하기" 버튼을 통해 관리자가 확인 후 수동으로 처리하는 방식으로 대체
