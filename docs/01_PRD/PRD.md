# 1. 프로젝트 개요

**프로젝트명 :** CampusEats (학식 말고 뭐 먹지?)

CampusEats는 대학생을 대상으로 학교 주변 식당 정보를 한곳에서 제공하는 반응형 웹 애플리케이션(Web Application)이다. 사용자는 식당 검색, 카테고리별 탐색, 메뉴 및 가격 조회, 리뷰 확인 기능을 이용하여 자신의 상황에 맞는 식당을 쉽게 찾을 수 있다. 또한 학생들이 직접 식당을 제보할 수 있어 데이터베이스를 지속적으로 확장할 수 있도록 설계한다.

# 2. 문제 정의

현재 학생들은 학교 주변 식당 정보를 네이버 지도, 카카오맵, SNS, 커뮤니티 등 여러 서비스에서 각각 찾아야 한다. 특히 학생들이 중요하게 생각하는 메뉴, 가격, 학생 리뷰, 학교 제휴 여부 등의 정보를 한곳에서 확인하기 어렵다. CampusEats는 이러한 정보를 통합 제공하여 학생들이 쉽고 빠르게 식당을 선택할 수 있도록 하는 것을 목표로 한다.

# 3. 프로젝트 목표

| 목표 | 내용 |

| 검색 편의성 | 검색 및 카테고리 필터를 통해 원하는 식당을 빠르게 탐색 |
| 학생 참여 | 리뷰 작성과 식당 제보 기능으로 사용자 참여 유도 |
| 운영 관리 | 관리자 페이지를 통해 식당 및 리뷰를 효율적으로 관리 |

# 6. 요구사항

## 6.1 기능 요구사항

- 회원은 회원가입 및 로그인을 할 수 있다.
- 사용자는 식당명 또는 메뉴명으로 식당을 검색할 수 있다.
- 카테고리를 이용하여 원하는 식당을 탐색할 수 있다.
- 식당 상세 페이지에서 메뉴, 가격, 위치, 리뷰를 확인할 수 있다.
- 로그인한 사용자만 리뷰를 작성할 수 있다.
- 사용자는 신규 식당 등록을 요청할 수 있다.
- 관리자는 식당, 리뷰, 회원, 제보를 관리할 수 있다.

## 6.2 시스템 요구사항

- 모바일 환경에 최적화된 반응형 UI를 제공한다.
- 학교 주변 식당 데이터를 기반으로 서비스를 제공한다.
- 사용자 인증은 Supabase Auth를 사용한다.
- 지도는 Kakao Map API를 사용한다.
- GitHub를 이용하여 버전 관리를 수행한다.

## 6.3 제약사항

- 개발 기간은 2주이다.
- AI(Vibe Coding)를 활용하여 개발한다.
- 리뷰 사진 및 AI 추천 기능은 MVP 범위에서 제외한다.
- 서비스 대상은 특정 대학 주변 식당으로 한정한다.

# 7. User Flow & Admin Flow

## 7.1 User Flow

```
앱 실행
  │
  ▼
메인 화면
  │
  ┌──────────────┴──────────────┐
  ▼                             ▼
식당 검색                    카테고리 선택
  │                             │
  └──────────────┬──────────────┘
                 ▼
              식당 목록
                 │
                 ▼
              식당 상세
                 │
   ┌─────────────┼─────────────┐
   ▼             ▼             ▼
메뉴 확인      리뷰 확인      위치 확인
                 │
                 ▼
            리뷰 작성 버튼
                 │
            로그인 여부 확인
                 │
           ┌─────┴─────┐
           │  미로그인  │
           └─────┬─────┘
                 ▼
               로그인
                 │
                 ▼
           리뷰 작성 완료
                 │
                 ▼
              상세 페이지
```

---

**식당 제보**

```
메인 화면
  │
  ▼
식당 제보
  │
  ▼
식당 정보 입력
(식당명 / 위치 / 카테고리)
  │
  ▼
제보 등록
  │
  ▼
관리자 승인 대기
```

사용자는 메인 화면에서 식당을 검색하거나 카테고리를 선택하여 원하는 식당을 탐색할 수 있다. 식당 상세 페이지에서 메뉴, 가격, 위치, 리뷰를 확인할 수 있으며, 리뷰 작성은 로그인한 사용자만 가능하다. 등록되지 않은 식당은 '식당 제보' 기능을 통해 관리자에게 등록을 요청할 수 있다.

## 7.2 Admin Flow

```
관리자 로그인
  │
  ▼
관리자 대시보드
  │
  ┌─────────────┼─────────────┬─────────────┐
  ▼             ▼             ▼             ▼
회원 관리     식당 관리     리뷰 관리     식당 제보 관리
  │             │             │             │
  ▼             ▼             ▼             ▼
회원 조회     식당 추가     리뷰 조회     제보 목록 조회
회원 관리     식당 수정     리뷰 삭제     승인 / 반려
              식당 삭제                     │
                                            ▼
                                      승인 시 DB 등록
```

관리자는 관리자 대시보드를 통해 회원, 식당, 리뷰 및 식당 제보를 관리한다. 사용자가 등록을 요청한 식당은 관리자의 승인 후 데이터베이스에 등록되며, 이를 통해 서비스의 식당 정보를 지속적으로 유지·관리한다.
