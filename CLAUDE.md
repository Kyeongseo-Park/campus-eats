## 나의 작업 원칙
- Phase가 끝날 때마다 브라우저에서 직접 확인 가능한 체크리스트를 반드시 제공한다
- 커밋과 푸시는 사용자가 명시적으로 "커밋해줘"라고 말하기 전까지는 하지 않는다

## 승인 요청 원칙
다음 세 가지 중 하나에 해당하면, 실행 전 반드시 한 줄로 설명하고 승인을 받는다:
1. 되돌리기 어려운 작업 (삭제, 강제 push, DB 구조 변경 등)
2. 외부에 공개되거나 남에게 영향을 주는 작업 (배포, PR 병합, 환경변수 변경 등)
3. 비밀 정보(.env, API 키, 토큰)와 관련된 작업

단순 조회/읽기(git status, ls, cat 등)나 로컬 개발 서버 실행(npm run dev) 같은
안전하고 되돌리기 쉬운 작업은 설명 없이 바로 진행해도 된다.
